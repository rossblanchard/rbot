# INFRA__google_drive_sync_integration__v1.0

## 1. Overview

This document defines the integration of Google Drive with the Ross Brain infrastructure using rclone on a Raspberry Pi host. The purpose of this integration is to enable persistent, remotely accessible storage for artifacts generated by AnythingLLM running inside Docker.

Authentication is handled via OAuth 2.0. Google account passwords are never stored on the Raspberry Pi.

---

## 2. Architectural Context

Host Environment:
- Raspberry Pi 4 (8GB)
- AnythingLLM running inside Docker
- Public access via rbot.rossblanchard.com

Storage Integration Model:

AnythingLLM (Docker)
    ↓
Mounted Host Volume (/gdrive)
    ↓
rclone Mount
    ↓
Google Drive (Cloud Storage)

Google Drive is mounted on the host and then exposed to the container via Docker volume mapping.

---

## 3. Installation and Configuration

### 3.1 Install rclone

```bash
sudo apt update
sudo apt install rclone -y
```

Verify installation:

```bash
rclone version
```

---

### 3.2 Configure Google Drive (Headless OAuth Flow)

Run:

```bash
rclone config
```

Configuration Steps:

1. Select `n` for New remote
2. Name: `gdrive`
3. Storage type: `drive`
4. Client ID: press Enter (default)
5. Client Secret: press Enter (default)
6. Scope: select default (full access) or limited scope as desired
7. Root folder ID: press Enter
8. Service account: press Enter
9. Auto config: select `n`

rclone will generate a URL.

Steps:
1. Copy URL from SSH session
2. Open in browser on local machine
3. Authenticate with Google account
4. Approve access
5. Copy authorization code
6. Paste code into SSH session

rclone stores a refresh token at:

```
~/.config/rclone/rclone.conf
```

Secure the file:

```bash
chmod 600 ~/.config/rclone/rclone.conf
```

---

### 3.3 Verify Connection

```bash
rclone lsd gdrive:
```

Expected result: List of Google Drive folders.

---

## 4. Mount Configuration

### 4.1 Create Mount Point

```bash
mkdir -p /home/ross/GoogleDrive
```

### 4.2 Mount Google Drive

```bash
rclone mount gdrive: /home/ross/GoogleDrive --daemon
```

Verify:

```bash
ls /home/ross/GoogleDrive
```

---

## 5. Artifact Directory

Create Ross Brain artifact directory:

```bash
mkdir -p /home/ross/GoogleDrive/RossBrain
```

All generated artifacts should be written to:

```
/home/ross/GoogleDrive/RossBrain/
```

---

## 6. Docker Integration

Update `docker-compose.yml` for AnythingLLM:

```yaml
volumes:
  - /home/ross/GoogleDrive:/gdrive
```

Restart container:

```bash
docker compose down
docker compose up -d
```

AnythingLLM can now write to:

```
/gdrive/RossBrain/
```

Files will automatically sync to Google Drive.

---

## 7. Security Considerations

- OAuth 2.0 used for authentication
- Google password is never stored
- Refresh token stored locally in rclone config
- Config file permissions restricted to owner (600)
- Tokens can be revoked via Google Account security settings

---

## 8. Operational Notes

- Mount must be restarted after reboot unless automated via systemd
- Network connectivity required for sync
- Large file writes may incur latency
